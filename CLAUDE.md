# AI-Powered Data Governance & Quality Lakehouse — Telecom Italia Milan CDR

## Project Purpose
Portfolio project demonstrating enterprise data engineering: Delta Lake Medallion architecture
with PySpark, PyTorch anomaly detection, Great Expectations-style data quality, and a
Claude-powered AI governance agent. Uses the Telecom Italia Milan CDR dataset schema.

## Dataset
**Telecom Italia Big Data Challenge** (Harvard Dataverse, doi:10.7910/DVN/EGZHFV)
- 100×100 grid of Milan city (10 000 grid squares)
- 10-minute CDR intervals, Nov 2013 – Jan 2014
- Fields: square_id, time_interval (Unix ms), country_code, sms_in, sms_out, call_in, call_out, internet
- Synthetic replica generated by `src/data/generate_synthetic.py` (500 squares × 7 days by default)

## Architecture
```
data/raw/cdr_records.csv
        ↓ Bronze (Delta)          — raw + audit columns
        ↓ Silver (Delta + Pandera) — cleaned, validated, quality flags
        ↓ PyTorch Autoencoder      — anomaly scoring (7-feature CDR model)
        ↓ Gold (Delta)             — hourly_grid_activity, daily_country_usage,
                                     grid_heatmap, anomaly_summary
        ↓ Claude AI Governance Agent → governance/reports/governance_report_<ts>.md
```

## Key Files
- `src/data/generate_synthetic.py`    — synthetic CDR generator (mirrors real TI dataset)
- `src/utils/spark_session.py`        — Delta Lake + PySpark session factory (singleton)
- `src/pipelines/bronze.py`           — raw CSV → Delta with metadata columns
- `src/pipelines/silver.py`           — type coercion, null handling, dedup, pandera validation
- `src/pipelines/gold.py`             — CDR aggregations (hourly_grid_activity, daily_country_usage, grid_heatmap)
- `src/models/autoencoder.py`         — PyTorch Autoencoder (7→32→16→4→16→32→7)
- `src/models/train.py`               — trains on clean CDR data, saves threshold + weights
- `src/models/inference.py`           — scores all CDR records, writes anomaly_scores to silver
- `src/quality/validators.py`         — pandera CDR schema + custom quality checks
- `src/quality/reporter.py`           — generates JSON + Markdown quality reports
- `src/agents/governance_agent.py`    — Claude API agent → governance docs + data catalog
- `flows/main_flow.py`                — Prefect orchestration of the full pipeline

## Environment
Requires `.env` with `ANTHROPIC_API_KEY`. See `.env.example`.

## Run Order
1. `make data`   — generate synthetic CDR (500 squares × 7 days ≈ 650k rows)
2. `make bronze` — ingest to Delta Lake
3. `make silver` — clean + validate
4. `make model`  — train anomaly detector
5. `make gold`   — aggregate CDR data
6. `make agent`  — generate AI governance report

Or: `make all`

## Conventions
- All Delta tables written to `data/{layer}/` as local Delta format
- Quality reports saved to `governance/reports/` as JSON + Markdown
- Lineage metadata saved to `governance/lineage/` as JSON
- Model artifacts saved to `src/models/trained/`
- All scripts callable as `python -m src.module_path`

## CDR Anomaly Types Detected
| Type | Description |
|------|-------------|
| `activity_spike` | Grid-interval with 15–50× normal activity (major event, concert, protest) |
| `dead_zone` | Grid with all-zero activity (infrastructure outage, no coverage) |
| `negative_activity` | Negative CDR metric value (data corruption / pipeline bug) |
| `future_timestamp` | Record timestamped beyond dataset window (ETL clock drift) |
| `null_activity` | All five activity metrics null (missing data, failed aggregation) |
